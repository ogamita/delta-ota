Ogamita Delta OTA

Ship the change, not the whole release.

Vanilla auto-updaters re-download the full payload at every release — 1 to 4 GB per user per month for a typical desktop application. Ogamita Delta OTA ships only the binary delta between two releases, typically ~1% of a full re-install. Users on flaky Wi-Fi, constrained corporate VPNs, and laptops that go on- and offline finally get to update reliably.

Why it matters

1 % of the bytes

A 2 GB monthly upgrade typically transfers ~20 MB. Lower bandwidth bill, faster updates, happier users.

Never breaks the install

Atomic switch-over: the agent flips to the new release only after the bytes have been downloaded, hashed, patched, re-hashed, and extracted. Failures leave the prior install untouched.

Zero-dependency client

Single static binary per OS. No DLL hell, no admin rights, no OpenSSL, no installer. Drop it next to your app.

Cryptographically signed

Every release manifest is Ed25519-signed and pinned to the publisher's public key. Tampered bytes are refused before a single file is written.

How it works

1. Publish

Your CI runs ota-admin publish. The server hashes the payload, deterministically packs it, and builds binary patches against every prior release.

2. Distribute

The agent fetches the signed manifest, picks the smallest viable patch, downloads it with streaming SHA-256 verification, and reconstructs the new release locally.

3. Switch over

The new distribution lands in a sibling directory. Once verified, an atomic symlink flip points current at it. Old material is pruned automatically.

The client side, in one command:

$ ota-agent install schms \
    --server https://ota.example.com \
    --install-token YOUR_TOKEN \
    --latest

installed schms 3.4.1

What you get

Server Linux x86_64 / arm64. Multi-arch container image. HTTP/JSON API. sendfile(2) for blob downloads. SQLite catalogue (PostgreSQL sub-phase). FS or S3 storage.
Client library Embeddable C-ABI shared library + static archive. Pure-Go internals, no cgo, no OpenSSL. libota.h header for Python, .NET, AutoLISP.
Client CLI Single-file ota-agent per OS, ~10 MB. install, upgrade, revert, doctor --recover, watch.
Operations Garbage collection, content-integrity verification, backup/restore scripts, audit log, install-token bulk issuance, recovery rollback to anchor releases.
